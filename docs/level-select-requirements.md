## 一、设计目标

### 1.1 核心体验定位

关卡选择界面的核心目的：**用美术和动效给玩家营造气氛与期待，增强代入感**。
它不只是一个功能选单，而是一个"世界入口"——玩家在此感受游戏的视觉基调，在关卡节点间游走时产生"我想打这关"的冲动。

### 1.2 设计原则

| 原则 | 含义 |
|------|------|
| **世界感优先** | 界面是一个"场所"而非"面板"。卫星轨道、能量节点、星空背景形成可信的科技世界观隐喻 |
| **信息密度克制** | 每次只展示一个章节（12 关），避免 48 关全铺导致的认知过载 |
| **状态可读** | 已完成/可挑战/已解锁/锁定四种状态必须在视觉上一眼可辨，不依赖文字标注 |
| **动效为叙事服务** | 所有动画不是为了炫技，而是强化"系统在运作"的感觉——粒子流动、节点呼吸、扫描线横扫 |

## 二、视觉隐喻与布局

### 2.1 隐喻：卫星轨道网络监控终端

玩家以**终端操作员的视角**审视一个轨道卫星网络：
- **4 条倾斜椭圆轨道** = 4 个章节（不同轨道代表不同能量层级/区域）
- **48 个节点** = 能量卫星，沿轨道排列
- **中央核心** = 反应堆/主控枢纽，为轨道提供能量
- **右侧信息卡** = 选中节点的数据面板
- **底栏进度条** = 本章能量激活程度

### 2.2 区域划分

```
┌─────────────────────────────────────────────┐
│  顶栏：返回按钮 · 章节标题（罗马数字+名称）· 设置按钮  │
├─────────────────────────────┬───────────────┤
│                             │               │
│   星空背景 + 章节背景图       │    信息卡      │
│   （12% screen 混合）        │  标题·描述     │
│                             │               │
│  ╭── 轨道区域 ──────────╮    │               │
│  │ 4 条椭圆轨道           │   │               │
│  │ ◀ 箭头 → 章节指示点   │   │               │
│  │ 12 个关卡节点         │   │               │
│  │ 中央核心（外圈+内点）   │   │               │
│  │ 扫描线（切换章节时横扫） │   │               │
│  │ 轨道粒子（流动绿色光点） │   │               │
│  ╰──────────────────────╯    │               │
│                             │               │
├─────────────────────────────┴───────────────┤
│  底栏：阶段进度 · 进度条 · X/12 · 进入关卡按钮    │
└─────────────────────────────────────────────┘
```

### 2.3 关卡节点布局逻辑

- 12 个节点沿当前章节轨道均匀分布
- 节点位置由配置文件定义，不同章节可复用同一套轨道坐标
- 间章节点（叙事关卡）穿插在轨道上，形态区别于普通节点（方形虚线边框）

## 四、动画规范

### 4.1 常驻动画

| 元素 | 效果 | 周期 |
|------|------|------|
| 轨道环（4 条） | 辉光呼吸，外层比内层慢 | 4s / 4.8s / 5.6s / 6.4s |
| 中央核心外圈 | 辉光 + 边框脉冲 | 2s |
| 中央核心内点 | 缩放（1× → 1.4×）+ 透明度跳动 | 1s |
| 已完成节点 | 辉光呼吸 | 2.5s |
| 可挑战节点 | 辉光 + 边框脉冲 | 1.6s |
| 间章节点 | 辉光脉冲 | 3s |
| 章节标题 | 文字辉光呼吸 | 3s |
| 进度条 | 辉光脉冲 + 光泽从左扫过 | 2s |
| 星空 | 星点随机闪烁 + 数学符号缓慢上浮 | 持续（rAF 驱动） |
| 轨道粒子 | 绿色光点沿椭圆轨道流动 | 持续（60fps） |

### 4.2 章节切换过渡动画

章节切换时执行三段式过渡，总时长约 2.1s，动作概要：

**第一阶段 — 停机（~500ms）**
- 所有关卡节点缩小淡出
- 间章节点缩小淡出
- 信息卡滑出右侧
- 轨道粒子涌向中央核心
- 导航箭头半透明 + 禁用
- 轨道环辉光减弱

**第二阶段 — 重构（~450ms）**
- 扫描线从上到下横扫
- 中央核心爆发辉光（外圈 60px + 内点放大 ×2.5）
- 章节标题文字故障闪烁（紫→青→绿）
- 新背景图开始淡入，旧图淡出（跨阶段）

**第三阶段 — 启动（~1200ms）**
- 新关卡节点交错淡入（间隔 60ms，带弹性）
- 间章节点交错淡入
- 章节标签淡入
- 轨道环辉光恢复
- 轨道粒子批量重生
- 信息卡滑入右侧
- 导航箭头恢复

**过渡期间规则**：
- 全部输入锁定（防止动画期间重复触发）
- 标题区域、节点区域、轨道粒子同时联动，不允许各自为政

### 4.3 入场动画

页面首次加载时的入场序列（非章节切换）：

| 元素 | 延迟 | 效果 |
|------|------|------|
| 顶栏 | 0s | 淡入 0.8s |
| 内容区域 | 0.2s | 上滑 1s |
| 底栏 | 0.4s | 淡入 0.8s |
| 关卡节点 | 内容区域就绪后 | 交错淡入 0.5s（间隔 60ms） |

### 4.4 Hover 动画（PC 端专属）

- 节点悬停：scale 1.15 + 外围波纹扩散
- 按钮悬停：沿用 ButtonHover 组件（上浮 + 荧光）

## 五、交互规范

### 5.1 导航

| 操作 | 输入 | 效果 |
|------|------|------|
| 上一章 | 点击 ◀ 箭头 / 键盘 ← | 切换到上一章节 |
| 下一章 | 点击 ▶ 箭头 / 键盘 → | 切换到下一章节 |
| 跳转指定章 | 点击章节指示点 ●●●● | 切换到对应章节 |
| 返回主菜单 | 点击顶栏返回按钮 | 跳转至主菜单场景 |
| 打开设置 | 点击顶栏设置按钮 | 弹出设置面板 |

### 5.2 关卡选择

| 操作 | 效果 |
|------|------|
| 点击 complete / challengeable / unlocked 节点 | 选中节点 → 信息卡更新 → 外围 outline 高亮 |
| 点击间章节点 | 选中节点 → 信息卡显示间章叙事摘要 |
| 点击 locked 节点 | 无响应 |
| 点击"进入关卡"按钮 | 跳转至战斗场景（需选中关卡有效） |

### 5.3 导航禁用规则

| 条件 | 禁用 |
|------|------|
| 当前为第一章 | ◀ 箭头禁用 |
| 当前为最后一章 | ▶ 箭头禁用 |
| 下一章未解锁 | ▶ 箭头禁用 |
| 过渡动画进行中 | 所有导航禁用 |

### 5.4 章节解锁规则

- 通关当前章节的 Boss 关卡 → 解锁下一章节
- DEMO 阶段所有章节可自由访问，方便开发调试

## 七、平台适配

### 7.1 分辨率

- 设计分辨率 1920×1080，适配模式 Fit Width
- 手机端锁横屏

### 7.2 移动端特殊处理

| 问题 | 策略 |
|------|------|
| 节点偏小（48px 在低分辨率 ≈ 17px） | 移动端检测后放大至 64px，触摸判定区扩至 80px |
| 箭头按钮偏小 | 移动端放大至 64px |
| 无 Hover | 跳过 hover 效果 |
| 无键盘 | 仅靠箭头 + 指示点导航 |
| 信息卡遮挡节点 | 移动端信息卡改为底部弹出条 |

### 7.3 性能预算

| 指标 | 预算 |
|------|------|
| 星空粒子 | ≤ 180 颗 + ≤ 20 个数学符号 |
| 轨道粒子 | ≤ 30 个同时存在 |
| 章节切换帧率 | ≥ 30fps（可接受短暂下降） |
| 首屏可用 | ≤ 2s（背景图异步不阻塞 UI） |
